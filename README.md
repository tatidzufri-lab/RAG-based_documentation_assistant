# Ассистент по документации ретрита «Познай себя»

## Описание проекта

Telegram-бот, который отвечает на вопросы клиентов о женском ретрите «Познай себя» в горах Кавказа. Бот находит нужную информацию в базе знаний и формулирует ответ на естественном языке — так, как это сделал бы живой консультант.

### Задача

Автоматизировать ответы на типовые вопросы потенциальных участниц ретрита: что входит в программу, как подготовиться, что взять с собой, для кого подходит программа и т.д. При этом бот должен корректно отклонять вопросы, выходящие за рамки его компетенции (например, запросы о маркетинговых стратегиях или внутренних данных).

### Решение

Система построена на подходе RAG (Retrieval-Augmented Generation) — сначала находит релевантные фрагменты из базы знаний, а затем на их основе генерирует ответ. Это позволяет боту отвечать точно по документам, а не выдумывать информацию.

Ключевая особенность — поддержка двух языковых моделей одновременно: ChatGPT (OpenAI) и GigaChat (Сбер). Пользователь прямо в Telegram может переключаться между ними и сравнивать качество ответов.

### Зачем так устроено

Проект совмещает две роли. С одной стороны, это рабочий бот для реального клиента — он уже отвечает на вопросы потенциальных участниц ретрита и снимает нагрузку с организатора. С другой стороны, в систему намеренно встроены инструменты, которые делают её прозрачной и удобной для интеграции: подробное логирование каждого запроса, встроенная статистика прямо в Telegram, экспорт логов в CSV, переключение между моделями одной командой и автоматическая сравнительная оценка по метрикам RAGAS. Благодаря этому можно в любой момент проверить качество ответов, сравнить поведение разных LLM и быстро адаптировать систему под новую предметную область или другого клиента.

## Как работает проект

1. **Подготовка базы знаний.** Текстовые документы о ретрите (описание, программа, правила, список вещей и др.) разбиваются на небольшие смысловые фрагменты и сохраняются в векторной базе данных.

2. **Обработка вопроса.** Когда пользователь задаёт вопрос в Telegram, система находит наиболее подходящие фрагменты из базы знаний с помощью семантического поиска.

3. **Генерация ответа.** Найденные фрагменты передаются языковой модели (ChatGPT или GigaChat — на выбор пользователя), которая формулирует связный и точный ответ.

4. **Кэширование.** Если точно такой же вопрос уже задавался — ответ берётся из кэша, что экономит время и ресурсы.

5. **Логирование.** Все запросы и ответы записываются в базу данных для последующего анализа качества работы системы.

### Что выдаёт проект

- **Telegram-бот** — основной интерфейс. Пользователь пишет вопрос, получает ответ. Команда `/mode` позволяет выбрать модель (ChatGPT или GigaChat), `/reindex` — обновить базу знаний после изменения документов.

- **Сравнительный отчёт RAGAS** — скрипт `evaluate_compare.py` прогоняет обе модели по 50 тестовым вопросам и оценивает качество ответов по трём метрикам: точность относительно контекста, точность контекста и релевантность ответа. Результаты выводятся в консоль и сохраняются в CSV-файл.

## Стек технологий

| Технология | Для чего используется |
|---|---|
| **Python 3.11** | Основной язык проекта |
| **OpenAI API** | Языковая модель ChatGPT (GPT-4o-mini) для генерации ответов и создания эмбеддингов (text-embedding-3-small) |
| **GigaChat API** | Языковая модель GigaChat от Сбера — альтернативный генератор ответов и эмбеддингов |
| **ChromaDB** | Векторная база данных для хранения фрагментов документов и семантического поиска по ним |
| **SQLite** | Локальная база данных для кэширования ответов и логирования запросов |
| **python-telegram-bot** | Библиотека для создания Telegram-бота |
| **RAGAS** | Фреймворк для автоматической оценки качества RAG-системы по метрикам Faithfulness, Context Precision и Answer Relevancy |
| **LangChain** | Используется для интеграции эмбеддингов OpenAI с фреймворком RAGAS |
| **python-dotenv** | Загрузка конфигурации (API-ключей) из файла `.env` |
| **NumPy, Pandas** | Вспомогательные библиотеки для работы с данными |

## Запуск

1. Установите зависимости:

   ```
   pip install -r requirements.txt
   ```

2. Создайте файл `.env` в корне проекта и заполните ключи:

   ```
   OPENAI_API_KEY=ваш_ключ_openai
   GIGACHAT_RQUID=ваш_rquid
   GIGACHAT_AUTH_KEY=ваш_ключ_gigachat
   TELEGRAM_BOT_TOKEN=токен_от_BotFather
   ```

3. Запустите бота:

   ```
   python run_bot.py
   ```

4. Для оценки качества ответов (необязательно):

   ```
   python evaluate_compare.py
   ```
